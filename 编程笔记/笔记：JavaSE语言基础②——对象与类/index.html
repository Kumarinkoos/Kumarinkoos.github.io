笔记：JavaSE语言基础②——对象与类

本篇章以Core Java为基础，记录自己学习Java以及这本书时候的笔记，供自己后续翻阅回顾用。

分为基础语法，面向对象，接口，容器，异常，泛型，反射，注解和I/O等篇章。

本小节为语言基础的对象与类篇，主要记录了面向对象，预定义类，自定义类等方面的笔记。

面向对象概述

大三特性：封装，继承，多态

面向对象

  面向对象程序设计（OOP），面向对象的程序由对象组成，每个对象包含对用户公开的特定功能部分和隐藏的实现部分。

类

  类是构造对象的模板或蓝图，由类构造对象的过程称为创建类的实例。

  封装：是处理对象的一个重要的概念。封装就是就是将数据和行为组合在一个包中，并对对象的使用者隐藏具体的实现方式。对象中的数据称为实例字段（实例变量），操作数据的过程称为方法。

  实现封装的关键在于，绝对不能让类中的方法直接访问其他类的实例字段。程序只能通过对象的方法与对象数据进行交互。

  继承：在Java中，所有的类都源于Object。所有类都扩展自这个Object。扩展一个已有的类时，这个扩展后的新类具有被扩展类的全部属性和方法（不继承构造方法）。通过扩展一个类来建立另外一个类的过程称为继承

  多态：

对象

  对象的三个主要特性：

      1.对象的行为：可以对对象完成哪些操作，或者可以对对象应用哪些方法。

      2.对象的状态：当调用那些方法时，对象会如何响应。

      3.对象的标识：如何区分具有相同行为和状态的不同对象。

  对象的行为是用可调用的方法来定义的。

  对象的状态的改变必须通过调用方法实现。

  每个对象都有一个唯一的标识。作为同一类的实例，每个对象的标识总是不同的，状态往往也会有差异。

类之间的关系

  在类之间，最常见的关系有：1.依赖（uses a）  2.聚合（has a）  3.继承（is a）

  依赖（uses a）：如果一个类的方法使用或操作另一个类的对象，我们就说一个类依赖于另一个类。我们应该尽可能地将相互依赖的类减至最少（解耦合）。即尽可能减少类之间的耦合。

  聚合（has a）：包容关系意味着类A的对象包含类B的对象。（即B是A的一个属性）

  继承（is a）：表示一个更特殊的类与一个更一般的类之间的关系。一般而言，如果类A扩展类B，类A不但包含从类B继承的方法，还会有一些额外的方法。

预定义类

  想要使用对象，首先必须构造对象，并指定其初始状态。然后对对象应该方法。

需要构造方法的Date类

  在Java中，要使用构造器（构造方法）构造新实例。构造器是一种特殊的方法，用来构造并初始化对象。

  构造器的名字应该和类名相同，想要通过构造器构造一个对象，需要在构造器前面加上一个new操作符。

其中new Date()为创建了一个对象，变量birthday是一个引用，里面存放的是对象的地址。

要重要的认识到一点：对象变量并没有实际包含一个对象，它只是引用一个对象。在Java中，任何对象变量的值都是对存储在堆内存中的某个对象的引用。包括new操作符的返回值也是一个引用。

  ※对象由new+构造器创建并存在堆内存中，而引用变量存在栈中，变量里存放的引用指向的对象的内存地址。

  可以显式地将对象变量设置为null，只是这个对象目前没有引用任何对象（编译能通过，但运行会有空指针异常NullPointException）。

  ※可以把Java中的对象变量看做类似于C++中的对象指针。

不需要构造方法的LocalDate类

  在创建LocalDate对象的时候，不需要调用构造器。应该使用静态工厂方法，它会代表你调用构造器。

调用这个方法会构造一个新的对象，并把初始化为构造这个对象时的日期。

  可以使用LocalDate.of()来构造指定日期的对象

更改器方法（setter）和访问器方法（getter）

  更改器方法能修改对象的状态，但不会产生新的对象。

  访问器方法只能访问对象而不修改对象的状态。

  ※因为封装完整性，每个类都应该有其各个属性的setter和getter，即使可能暂时用不着也要写出来。

自定义类

构造器

  构造器与其他方法有有个重要的不同，构造器总是结合new运算符调用。不能对一个已经存在的对象调用构造器来达到重新设置实例字段的目的。

  构造器与类同名；每一个类可以有一个以上的构造器；构造器可以有多个参数；构造器没有返回值；构造器总是伴随着new一起调用。

  ※不要再构造器中定义与实例字段同名的局部变量。但在构造器的参数列表里可以使用同名

这里的this.name指的是类的实例字段，类似于对象.的方式。

var

  在Java10中，如果可以从变量的初始值推导出它们的类型，那么可以用var关键字声明局部变量，而不需指定类型。

  ※注意var关键字只能用于方法中的局部变量。参数和字段（成员变量）的类型必须声明。

null

  一个对象变量包含一个对象的引用，或者包含一个特殊值null，null表示没有引用任何对象。如果对null值应用一个方法，将会产生一个NullPointerException异常（空指针异常）。这是一个很严重的错误，为了解决这种错误，有两种解决方法：

  1：将null值转换为一个适当的非null值。

在Java9中Object类提供了一个便利的方法。

  2：直接拒绝null参数。

如果有人用了null构造对象，就会产生一个空指针异常，异常报告会提供这个问题的描述，以及报告会准确地指出问题所在的位置。（抛出的异常会显示”The name cannot be null”）

封装

  想要获得或设置实例字段的值，就需要提供三项内容：1.一个私有的数据字段（private）2.一个公共的字段访问器方法（getter）3.一个公共的字段更改器方法（setter）。

  注意不要编写返回可变对象引用的访问器方法。

Date类是可变类，返回了可变类后，接收Date类的变量d和Employee对象调用hireDay指向同一个Date对象了。那么对变量d的改变就能修改Date对象的私有状态了，这就破坏了封装性。如果需要返回一个可变对象的引用，首先应该对它进行克隆。

私有方法

  有时，可能希望将一个计算代码分解成若干个对立的辅助方法，通常这些辅助方法不应该成为公共接口的一部分。这时最好的方法是将这些辅助方法设置为私有方法。在Java中要实现私有方法只需要将关键字public改为private即可

final实例字段

  可以将实例字段定义为final。这样的字段必须在构造对象时初始化。也就是说，必须确保在每一个构造器执行之后，这个字段的值以及设置，并且以后不能再修改。final修饰符对于基本类型和不可变类的字段尤其有用。

静态变量与静态方法

  静态字段和静态方法都是由static修饰的。使用类名.来访问。

静态字段

  静态变量：成员变量使用static修饰后，将属于类，而不属于任何单个对象。

  静态常量：静态变量实际使用不多，更多为静态常量，即使用static和final同时修饰。

静态方法

  在调用时不需要对象，使用类名.的方式来调用。静态方法因为不需要对象，所以不能访问实例字段，但静态方法可以访问静态字段。

  ※在Java中虽然可以使用对象来调用静态方法，但因为这样容易造成混淆，所以建议使用类名来调用静态方法。

  在以下两种情况下可以使用静态方法：1.方法不需要访问对象状态，因为它需要的所有参数都通过显式参数提供。2.方法只需要访问类的静态字段。

  

  工厂方法：静态方法还有另外一种常见的用途。类似LocalDate，NumberFormat的类使用静态工厂方法来构造对象（工厂方法代表程序员调用构造方法）

为什么这些类不使用构造器呢：1.无法命名构造器。构造器名字必须和类名相同。但是这里希望有两个不同的名字分别得到不同的实例。2.使用构造器时，无法改变所构造对象的类型。而工厂方法实际将返回一个子类。而非调用这个方法的类 （存疑，可以直接使用向上转型而不必使用工厂方法）

main方法

  main方法前也是有static修饰的。其实可以在每个类中都设置main方法用于对类的测试（单元测试）。

方法参数

  程序设计中将参数传递给方法有：按值调用（表示方法接收的是调用者提供的值）；按引用调用（表示方法接收的是调用者提供的变量地址）；还有一种按名调用。在Java程序中，是采用按值调用的。也就是说吗，方法得到的是所有参数值的一个副本，方法的调用不能修改给它的任何参数变量的内容。

  对于基本数据类型：x初始化为percent值的副本，x乘3后，但percent仍然没变，方法结束后x释放，percent仍为10。所以一个方法不能修改基本数据类型。

  对于引用数据类型：x初始化为harry的一个副本，这里就是一个对象的引用，x副本引用和harry引用同时指向Employee，所有对引用调用方法，就能改变对象的状态，方法结束后x释放，但harry指向的对象的状态已经被改变。

  对于交换对象：a，b初始化为alice和bob的副本，这里就是这两个对象的引用，a副本和alice指向同一个对象，b副本和bob指向同一个对象，现在将a和b进行交换，a副本和bob指向同一个对象，b副本和alice指向同一个对象，方法结束后，释放a和b，alice和bob仍然指向调用方法前的对象。

※总结：方法不能修改基本数据类型的参数（即数值型和布尔型）。

    方法可以改变对象参数的状态。

    方法不能让一个对象参数引用一个新的对象。

对象构造

重载（overlord）

  如果多个方法有相同的名字，不同的参数，便会出现重载。编译器必须挑选出具体条用哪个方法。它用各个方法首部中的参数类型与特定方法调用中所使用的的值的类型进行匹配，来选出正确的方法。（这个查找的过程被称为重载解析）

在Java中允许重载任何方法，要完整地描述一个方法，需要指定方法名以及参数类型。（这叫作方法的签名）。返回类型不是方法签名的一部分，也就是说不能有两个名字相同，参数类型和数目相同，却又不同返回类型的方法。

构造器

  如果写一个类时没有编写构造器，系统会默认提供一个无参数构造器。这个构造器将所有的实例字段设置为默认值（数值为0，布尔为false，对象引用为null）。

  如果一个类中手动提供了一个有参数构造器，系统默认提供的无参数构造器会消失。那么创建对象时调用无参构造器将是不合法的。（一般建议无参数构造器和有参数构造器都手动添加）。

  在构造器中，参数变量名经常会和实例字段名相同。但实例字段名前面其实是有this.（指示隐式参数），也就是所构建的对象。

调用构造器时，将传进来的name变量和id变量的值赋值给name实例字段和id实例字段。（this.是可以省略的，编译器编译时会自动填充this.）

调用另一个构造器

  如果构造器的第一句语句形式是this(…)，这个构造器将调用同一个类的另一个构造器，具体调用哪个构造器，看传进来的参数类型和参数个数。

初始化块

  初始化数据字段除了在构造器中设置值和在声明中赋值，还有使用初始化块。

  非静态代码块，在创建对象时运行且在构造器之前运行。

  静态代码块，在类第一次加载时运行。并且使用static修饰。

  静态代码块为程序员提供了一个时机，即加载类的时机。非静态代码块也为程序员提供了一个时机，即创建对象的时机。

  ※从类加载到初始化：

    创建对象时，先加载对象的类，加载时执行类的静态代码块，对象创建，执行费静态代码块，最后执行构造器。如果构造器的第一行调用了其他构造器则调用所对应的构造器。

包

  使用包的主要原因是为了保证包名的绝对唯一性，通常使用因特网域名逆序的形式作为包名，然后对于不同的工程师使用不同的子包。例如：com.kumarinko.core.Employee。

导包

  我们可以采用两种方式访问另一个保重的公共类。

    第一种方式就是使用完全限定名，就是类名前面有包名。

    第二种方式就是使用import语句。

使用java,time.*的语法比较简单，对代码的规范也没有任何负面影响。不过，如果能够明确地指出所导入的类，代码的读者能更加准确地知道使用了哪些类。

※需要注意的是，只能使用星号导入一个包，而不能使用import java.或者import java..*来导入以java为前缀的所有包。

  如果发生命名冲突的时候，需要特别注意。例如java.util和java.sql包都有Date类。

这时候会出现编译错误，编译器无法知道哪个是需要用的包，这时候将特定的包导进来就好，但如果两个Date类都要使用的话，那只能在每个类名的前面加上完整的包名了。

静态导入

  静态导入允许导入静态方法和静态字段，而不只是类

建包

  package是java中的包机制，要想将类放入包中，就必须将包的名字放在源文件的开头，即放在定义这个包中各种类的代码前。

  如果没有在源文件中放置package语句，这个源文件中的类就属于无名类。

  ※编译器在编译源文件的时候不检查目录结构。即使源文件不在子目录下，也可以进行编译。如果它不依赖与其他包，就可以通过编译，但最终程序运行不了，因为如果包与目录不匹配，虚拟机就找不到类。

访问权限

访问权限 含义 本类 本包类 非本包子类 非本包非子类
public 公共的 是 是 是 是
protected 保护访问权限 是 是 是 否
default 包访问权限 是 是 否 否
private 私有的 是 否 否 否

文档注释

  JDK有个叫做javadoc的工具，它可以由源文件生成一个HTML文档。文档注释以 /** 开头，以 */ 结束。注释中的标记以@开始。

  文档注释的第一句话应该是一个概要性的句子。javadoc会将这些句子抽取来生成概要页。

通用注释

※需要注意的是，在使用@see和@link的时候一定要用#来分隔类名与方法名，或者类名与变量名。

@see标记后加< 可以超链接到任何URL。

类注释

  类注释必须放在import语句之后，类定义之前。

方法注释

  每个方法注释必须放在所描述的方法之前。除了通用标记以外，还可以有一下标记：

字段注释

  对公共的字段（通常是静态常量）建立文档

包注释

  包注释需要在每一个包目录中添加一个单独的文件，可以有以下两个选择：

  1.提供一个名为package-info.java的Java文件。这个文件必须包含一个初始的以 /** 和 */ 界定的Javadoc注释，后面是一个package语句。它不能包含更多的代码或注释。

  2.提供一个名为package.html的HTML文件。会抽取标记< body >…< /body >之间的所有文本。

类设计技巧

一定要保证数据私有。

  绝对不要破坏封装性。保持实例变量的私有化。

一定要对数据进行初始化。

不要在类中使用过多的基本类型。

不是所有的字段都需要单独的访问器和更改器。

分解有过多职责的类。

类名和方法名要能体现它们的职责。

  类名应该是一个名词，或者是有形容词修饰的名词，亦或是有动名词修饰的名词。

  方法名应该是一个动词。遵循标准惯例：访问器方法用小写get开头，更改器方法用小写的set开头。

优先使用不可变类。
