总结：JavaSE——Object类及方法

发表于 2020-08-07 分类于 编程心得 阅读次数：
本文字数： 8.5k 阅读时长 ≈ 8 分钟

Object类是Java所有类的根基类，在Java中每个类都是由它扩展而来的。本篇为目前为止学习的与Object类有关的学习总结。

Object类简介

public class Object

  Object类是java.lang包下的一个类。Object类是Java所有类的最终祖先，Java中的每个类都是由它扩展而来。如果在类的声明中未使用extends关键字指明其父类，则默认父类为Object类。后面将总结Object类中常用的方法。

toString()

String toString()

描述：返回对象的字符串表示形式。

源代码：

  toString()方法用于返回表示对象值的字符串，Object类的toString()方法返回一个字符串，该字符串由“类名+@+此对象哈希码的无符号十六进制”表示组成。

  toString()方法非常常见，只要对象与一个字符串通过操作符“+”连接起来，Java编译就会自动调用toString()方法。

  System.out.println()方法的参数为一个对象时，它将自动调用该对象的toString()方法。

  强烈建议自定义类重写Object的toString()方法

  一般在重写toString()方法的时候，最好通过getClass().getName()获得类名的字符串，而不要将类名硬编码到toString()方法中。如果在父类中重写了toString()方法，那么子类主要调用super.toString()就可以了

  IDEA自动生成的toString()方法。

finalize()

protected void finalize()

描述：当垃圾收集确定不再有对该对象的引用时，垃圾收集器在对象上调用该对象。

※从Java9开始已弃用。最终确定机制本质上存在问题。

源代码：

  垃圾回收器（Garbage Collection），也叫GC。垃圾回收器主要有以下特点：

    1.当对象不再被程序使用时，垃圾回收器会将其回收。

    2.垃圾回收是在后台运行的，我们无法命令垃圾回收器马上回收资源，但我们可以告诉他尽快回收资源（System.gc()）。

    3.垃圾回收器在回收某个对象的时候，首先会调用该对象的finalize方法。

    4.GC主要针对堆内存。

    5.finalize方法运行时间不确定，如果执行此方法出现错误，程序不会报错，，仍然继续运行。

  虽然不建议使用这个方法，但这个方法给程序员提供了一个时机，即对象被GC回收的时机。

equals()

public boolean equals(Object obj)

描述：指示一些其他对象是否等于此。

源代码：

  在Object类中，这个方法将判断两个对象引用是否相等（==对引用是检测的是里面的内存地址）。不过由于我们经常需要给予状态检测对象的相等性（即对象的属性是否相等），如果两个对象有相同的状态，才认为这两个对象是相等的，所以基本上我们需要重写equals()方法。

  Java语言要求规范equals()方法具有下面的特性：

    1.自反性：对于任何非空引用x，x.equals(x)应该返回true。

    2.对称性：对于任何引用x和y，当且仅当y.equals(x)返回true时，x.equals(y)返回true。

    3.传递性：对于任何引用x，y，z，如果x.equals(y)返回true，y.equals(z)返回true，x.equals(z)也应该返回true。

    4.一致性：如果x和y引用的对象没有发生变化，反复调用x.equals(y)应该返回相同的结果。

    5.对于任意非空引用x，x.equals(null)应该返回false。

  在继承和equals()中会存在一些问题（比较的两个对象可能不是一个类或者为父子类）：如果子类能够拥有自己的相等概念，则对称性需求将强制采用getClass()进行检测。

  如果由父类决定相等性概念，那么就可以使用instanceof检测，这样可以在不同子类的对象之间进行比较。

  在子类中定义equals方法时，首先调用父类的equals。如果检测失败，对象就不可能相等。如果父类中的字段都相等，就需要比较子类中的实例字段。

hashCode()

public int hashCode()

描述：返回对象的哈希码值。

源代码：

  源代码中的native表明了这个方法是调用了C++的代码。

  散列码（hash code）是由对象导出的一个整型值。散列码是没有规律的，所有两个不同的对象的散列码基本不会相同。

  由于hashCode()方法定义在Object类中，因此每个对象都有默认的散列码，其值由对象的存储地址得出。

  字符串的散列码是由内容导出的，所有最终指向同一个字符串的对象的散列码是相同的。
